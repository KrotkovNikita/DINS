Доброго времени суток.

В данном репозитории находится UML-диаграмма с описанием предоставленной базы данных, а также отдельные папки для каждого отчета, которые содержат SQL-запросы и скрины с результатами их выполнения.

*Отчет №2 "Top 10: Most active users" отображает пользователей с самым большим количеством звонков. Эта реализация показалась мне самой логичной для термина "активные пользователи". При необходимости, возможно отображение пользователей с самой длительной общей продолжительностью звонок - это не составит большого труда.
*Отчет №3 "Top 10:Users with highest charges" не содержит разделения по дням, поскольку данные представленны по звонкам, только одного дня.
